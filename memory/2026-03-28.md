# 2026-03-28 - Telegram Sudo Permissions Issue

**Time:** Saturday, 17:00-17:09 GMT+9  
**User:** Jochem - Aethir Enterprise (Telegram)  
**Interaction:** Troubleshooting sudo command execution

## What Happened
User tried to run `sudo apt update && sudo apt upgrade -y` through me on Telegram. The command failed with an error about missing approval ID.

## Root Cause
When I attempted to trigger the sudo approval system, I discovered that **elevated permissions are not configured for Telegram** in the OpenClaw configuration:

```
Error: elevated is not available right now (runtime=direct)
Failing gates: allowFrom (tools.elevated.allowFrom.<provider>)
```

## Options Presented
1. **Configure Elevated Permissions** - Requires admin to edit OpenClaw config to enable `tools.elevated.allowFrom.telegram: true`
2. **Use Different Interface** - Use web UI, Discord, etc. if already configured with sudo
3. **Run Commands Directly** - User executes `sudo apt update && sudo apt upgrade -y` on the machine
4. **SSH Access** - If SSH configured in TOOLS.md, connect via SSH to run sudo

## Resolution
User tried Option 1 but encountered errors in configuration. **Option 3 worked** - they ran the commands directly on the machine.

## Learnings
- OpenClaw's elevated permissions are channel-specific and require explicit configuration
- Telegram interface currently does NOT have sudo access enabled
- When sudo fails due to missing approval ID, the real issue is usually missing channel permissions, not the approval mechanism itself
- Direct command execution remains a reliable fallback when agent permissions are restricted
- User is technical and prefers understanding root causes over workarounds
